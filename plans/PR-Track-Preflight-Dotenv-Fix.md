## Mechanism

`scripts/smoke_content_ops_faq_saas_demo_route_e2e.py` calls `_load_dotenv_files()` from `_build_parser()`, which runs `load_dotenv(ROOT / ".env")` then `load_dotenv(ROOT / ".env.local", override=True)`. The failing test pops the required env vars (`ATLAS_API_BASE_URL`, token, account, ...) from the subprocess env to simulate missing inputs and expects exit code 2. On a checkout whose `ROOT/.env` holds live values, the subprocess reloads them, preflight passes, and the script exits 0, so `assert completed.returncode == 2` fails. CI passes because its clean checkout has no live `.env`. The recommended fix is an `ATLAS_DISABLE_DOTENV=1` (or `--no-dotenv`) opt-out the test sets; real runs keep loading dotenv unchanged. This PR records that; it does not implement it.

## Intentional
