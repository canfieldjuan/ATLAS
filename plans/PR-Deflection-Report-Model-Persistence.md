# PR-Deflection-Report-Model-Persistence

## Why this slice exists

Epic #1588's structured-report path needs paid reports to carry persisted
`deflection.v1` model data so old reports can be re-rendered by future web/PDF
surfaces without rerunning the FAQ pipeline. #1596 added `report_model` to
generated artifacts and #1598 added the registry, but the paid-report store
boundary still treats the model as incidental JSON inside `artifact`.

The root cause is not a missing Postgres column: `save_report(...)` already
persists the full artifact JSON. The remaining gap is that persistence is not
intentional or compatibility-guarded. A stored artifact with a current
`report_model` should round-trip through both stores, while legacy artifacts
without a model and future/unsupported schema versions should degrade to an
explicit absent model instead of crashing renderers.

## Scope (this PR)

Ownership lane: content-ops/deflection-launch-readiness
Slice phase: Vertical slice

1. Add a store-boundary helper that projects a safe stored
   `deflection.v1` model payload from a persisted paid-report artifact.
2. Expose that projection on `DeflectionReportAccessRecord` so future
   renderers can consume persisted model data without reparsing or rerunning the
   pipeline.
3. Prove `report_model` survives in-memory and Postgres store save/load
   round-trips.
4. Prove legacy artifacts without `report_model`, unsupported schema versions,
   and malformed section entries degrade safely.
5. Document the persisted-model compatibility rule in the frontend contract.
6. Archive the now-merged #1598 plan doc and refresh `plans/INDEX.md`.

## Mechanism

`deflection_report_access.py` gains a small projection helper at the storage
boundary:

- read `artifact["report_model"]` only when it is a mapping;
- accept only the current `deflection.v1` schema version;
- normalize summary/sections to plain JSON-compatible dictionaries;
- keep unknown section IDs so older consumers can skip them by surface;
- skip malformed section entries rather than failing the whole paid record.

`DeflectionReportAccessRecord.report_model()` exposes that projection. The raw
artifact remains unchanged so existing `/artifact` responses and delivery paths
continue to return exactly the stored payload after payment unlock.

## Intentional

- This slice does not add a new Postgres column. The artifact JSONB column is
  already the source of truth for the full paid artifact; duplicating the model
  into a second column would introduce drift.
- Unsupported schema versions return `None` rather than attempting best-effort
  coercion. Future renderers can decide whether to support a new schema, but
  the storage boundary should not guess.
- The helper validates shape enough for safe renderer selection, not full nested
  section semantics. Section-specific tests continue to own nested data
  correctness.
