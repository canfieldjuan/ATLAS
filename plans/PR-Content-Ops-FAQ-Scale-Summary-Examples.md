# PR-Content-Ops-FAQ-Scale-Summary-Examples

## Why this slice exists

PR-Content-Ops-FAQ-Scale-Smoke-Run-Summary added the top-level
FAQ run-summary field to scale-smoke artifacts. The checked-in large-run example
artifacts still show the older shape, so an operator comparing a fresh
1,000-row run against the examples would not see the new FAQ health rollup.

This slice refreshes those examples and the README guidance without changing
FAQ generation behavior.

## Scope (this PR)

Ownership lane: content-ops/faq-generator

1. Add the top-level FAQ run-summary field to the density-limited scale-smoke
   example.
2. Add the top-level FAQ run-summary field to the output-check-failure
   scale-smoke example.
3. Update README guidance to point operators at both the input profile and FAQ
   run-summary blocks.
4. Extend existing example tests so the current artifact shape stays pinned.

## Mechanism

The examples are static scale-smoke-shaped JSON fixtures. This slice adds the
same top-level FAQ run-summary block that the wrapper now writes and updates
the existing test that compares density-limited versus output-check-failure
examples.

No script or generator code changes are needed because the runtime behavior
already shipped in the prior slice.

## Intentional

- Static examples only. The scale-smoke wrapper already writes this field.
- The values are representative, not generated by a live archive fetch in this
  slice.
- No hosted UI changes; this keeps the operator runbook aligned with CLI
  artifacts first.

## Deferred

- Hosted UI display for the FAQ run-summary block remains separate.
- Parked hardening for this slice: none. The hardening tracker has no entries for this
  lane yet.
