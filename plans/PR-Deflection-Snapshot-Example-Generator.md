# PR-Deflection-Snapshot-Example-Generator

## Why this slice exists

PR #1803 made the backend snapshot projection contract registry-derived, but left
one deferred hardening gap: the committed frontend snapshot example is still a
hand-edited artifact. The current doc test compares that file to producer output,
which catches drift after the fact, but there is no backend-owned command that
regenerates the exact file or fails when it is stale.

Root cause: the contract guard lives at the final assertion layer while the
example artifact itself has no owned generation/preflight step. This PR fixes the
root for the snapshot example by making the committed JSON derivable from the
backend producer and by proving the check path fails on stale content.

## Scope (this PR)

   producers that feed the report contract tests
   rather than hand-written JSON.
3. Add tests proving the generated payload matches the committed example and
   that check mode exits non-zero for stale or missing output.

## Mechanism

The new script owns the representative deflection input rows used for the
frontend snapshot example. It builds the FAQ result with the real producer,
wraps it in the deflection report artifact, projects the free snapshot with the
same top-N cap used by the committed example, and serializes that payload with
stable JSON formatting. Normal mode writes the file. Check mode compares the
current file bytes to the generated bytes and exits 1 without writing when they
differ. Tests call the script helpers directly and the CLI entrypoint so both
the generation path and stale-file detector are exercised.

## Intentional

- This slice generates only the unpaid snapshot example. The full paid
  deflection report example can use the same pattern later, but adding both
  would widen the diff beyond the #1803 deferred snapshot hardening.
- The sample rows remain synthetic and deterministic; no live support-ticket
  CSV, database, or portfolio repo dependency is introduced for a docs contract
  fixture.

## Deferred

- Generate the full paid deflection report example with the same backend-owned
  pattern after this snapshot generator proves the command/test shape.
- Generate frontend/portfolio types from the backend contract registry; this
  slice only removes hand-edited drift from the committed snapshot JSON.

Parked hardening: none.
