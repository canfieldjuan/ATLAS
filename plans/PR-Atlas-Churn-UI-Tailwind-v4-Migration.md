# PR-Atlas-Churn-UI-Tailwind-v4-Migration

## Why this slice exists

Dependabot opened `#1781` to bump `atlas-churn-ui` from Tailwind 3.4.19 to
4.3.1. The manifest-only bump could not build because Tailwind v4 moved its
PostCSS plugin out of `tailwindcss` and removed the old
`@tailwind base/components/utilities` CSS entrypoint. Automated review also
flagged the Tailwind v4 `outline-none` behavior change: existing
`focus:outline-none` usages need `focus:outline-hidden` to preserve the old
forced-colors focus outline behavior.

This exceeds the 400 LOC soft cap because the lockfile churn is generated by
the Tailwind 3 to 4 dependency graph and the required `@tailwindcss/postcss`
adapter. The hand-authored migration is limited to package/config/CSS files and
a mechanical Tailwind utility rename.

## Scope (this PR)

Ownership lane: dependency-maintenance
Slice phase: Production hardening

1. Keep Dependabot's `tailwindcss@4.3.1` upgrade for `atlas-churn-ui`.
2. Add the Tailwind v4 PostCSS adapter package required by the existing Vite
   build path.
3. Migrate the `atlas-churn-ui` CSS entrypoint from removed Tailwind v3
   directives to the Tailwind v4 import.
4. Replace `focus:outline-none` with `focus:outline-hidden` in
   `atlas-churn-ui/src` so high-contrast/forced-colors focus indicators keep
   the old hidden-outline behavior.
5. Leave other UI packages and visual/source refactors out of this slice.

### Review Contract

Acceptance criteria:

- `atlas-churn-ui/postcss.config.js` uses `@tailwindcss/postcss`, not
  `tailwindcss` directly, as the PostCSS plugin.
- `atlas-churn-ui/src/index.css` no longer uses the removed Tailwind v3
  `@tailwind base`, `@tailwind components`, or `@tailwind utilities`
  directives.
- `atlas-churn-ui/src` no longer contains `focus:outline-none`.
- The local `atlas-churn-ui` CI-equivalent path installs cleanly, reports no
  high severity npm audit findings, passes tests, and builds.
- No package outside `atlas-churn-ui` changes.

Affected surfaces:

- `atlas-churn-ui` package dependency graph.
- `atlas-churn-ui` PostCSS/Tailwind build path.
- `atlas-churn-ui` global CSS entrypoint.
- `atlas-churn-ui` focus-outline utility classes.

Risk areas:

- Tailwind v4 output differences could affect visual styling; this slice proves
  the package build/test path and preserves the documented outline behavior.
- The generated Dependabot body no longer describes the full repair because this
  PR now includes source/config migration files.

## Mechanism

Tailwind v4 no longer allows PostCSS to load `tailwindcss` directly as the
plugin. The migration adds `@tailwindcss/postcss@4.3.1` and changes
`postcss.config.js` to load that adapter while leaving `autoprefixer` in place.

Tailwind v4 also replaces the old three-directive CSS entrypoint with a single
`@import "tailwindcss";`. The existing global and prose styles remain after that
import.

The outline migration is mechanical: `focus:outline-none` becomes
`focus:outline-hidden`, matching Tailwind's v4 upgrade guidance for preserving
the old forced-colors behavior.

## Intentional

- The slice does not migrate other Tailwind packages; each Dependabot PR has its
  own package-specific blockers.
- No visual restyling is attempted here. The build path and documented utility
  rename are the required fixes for this dependency/security PR to become
  reviewable.
- `autoprefixer` remains in `postcss.config.js` because it was already part of
  the package pipeline and removing it is not required for the Tailwind v4
  repair.

## Deferred

- Visual preview smoke for `atlas-churn-ui` after CI produces a clean preview
  signal.
- Separate Tailwind v4 migrations for `atlas-ui` and `atlas-intel-ui` stay in
  their own Dependabot lanes.
- `atlas-mobile` NativeWind/Tailwind migration remains separate because it is
  not safe as a dependency-only Tailwind v4 bump.

Parked hardening: none.
