# PR-Deflection-Snapshot-Example-Generator

## Why this slice exists

PR #1803 made the backend snapshot projection contract registry-derived, but the
committed frontend snapshot example still had no backend-owned command that
regenerates the file or fails when it is stale.

Root cause: the contract guard lives at the final assertion layer while the
example artifact has no owned generation/preflight step. This PR fixes that root
by deriving the committed JSON from the backend producer.

## Scope (this PR)

Ownership lane: deflection/full-report-actionability
Slice phase: Production hardening
Max files: 7

1. Add a backend-owned generator/check command for the frontend deflection
   snapshot example JSON.
2. Move the snapshot example fixture source into that command so the committed
   example is produced from the same FAQ, report-artifact, and snapshot
   producers that feed the report contract tests.
3. Add tests proving the generated payload matches the committed example and
   that check mode exits non-zero for stale or missing output.
4. Enroll the generator check and test in extracted PR CI so stale examples are
   caught by the gate, not only by manual local verification.

### Review Contract

- Acceptance criteria:
  - [ ] The snapshot example check command passes

  - [ ] The command writes the canonical pretty-printed JSON when run without

  - [ ] A stale output file makes `--check` fail closed with a non-zero exit and
        a useful message.
  - [ ] Existing contract-doc tests still prove the snapshot example is unpaid

  - [ ] Extracted PR CI runs the generator check and the generator failure-path

- Reviewer rules triggered: R1, R2, R5, R10, R12, R14. Boundary-probe required

## Mechanism

The new script owns the representative deflection rows, builds the FAQ result
with the real producer, wraps it in the deflection report artifact, projects the
free snapshot with the committed top-N cap, and serializes stable JSON. Normal
mode writes the file. Check mode compares current bytes to generated bytes and
exits 1 without writing when they differ. Tests exercise helpers and the CLI.
The extracted CI runner invokes check mode and runs the generator tests under
the enrolled content-ops test namespace.

## Intentional

- This slice generates only the unpaid snapshot example; the full paid example
  can use the same pattern later.
- The sample rows remain synthetic and deterministic; no live CSV, database, or portfolio dependency is introduced.

## Deferred

- Generate the full paid deflection report example with the same backend-owned
  pattern after this snapshot generator proves the command/test shape.
- Generate frontend/portfolio types from the backend contract registry; this
  slice only removes hand-edited drift from the committed snapshot JSON.

Parked hardening: none.
