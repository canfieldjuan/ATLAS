# PR-Security-Disclosure-Policy

## Why this slice exists

Issue #1656 M8 calls out that Atlas has no first-party vulnerability disclosure
policy and no RFC 9116 well-known security.txt endpoint. That leaves
researchers and customers without an obvious reporting path while this security
hardening lane is tightening data deletion, token handling, and incident
surfaces.

Root cause: the repo has scanner/guardrail documentation, but no public
disclosure contract or runtime metadata route. This PR fixes that root for the
Atlas repo by adding the policy document and serving `security.txt` directly
from the FastAPI app.

## Scope (this PR)

Ownership lane: security/hardening-1656
Slice phase: Production hardening

1. Add `SECURITY.md` with Atlas vulnerability reporting expectations,
   researcher guidance, and initial response targets.
2. Add the well-known security.txt route with RFC 9116 fields for Contact,
   Policy, Preferred-Languages, and dynamic Expires.
3. Add focused tests for the generated security.txt body and served route.

### Review Contract

- Acceptance criteria:
  - [ ] `SECURITY.md` exists at the repo root and names the vulnerability
        reporting path.
  - [ ] The well-known security.txt route returns text/plain without auth.
  - [ ] The security.txt body includes Contact, Policy, Preferred-Languages,
        and Expires fields.
  - [ ] Expires is generated dynamically so the route does not ship a stale
        fixed date.
- Affected surfaces: public metadata route, repository security policy.
- Risk areas: public disclosure accuracy, route precedence with the UI static
  mount.
- Reviewer rules triggered: R1, R2, R5, R12, R14.

## Mechanism

`atlas_brain/main.py` exposes a small unauthenticated security.txt
route before the catch-all UI static mount. The response is generated by a
helper that computes `Expires` at request time, keeping the metadata valid
without recurring date-only maintenance. The policy URL points at the root
`SECURITY.md` file on GitHub.

## Intentional

- This PR does not touch `atlas-portfolio`; the issue explicitly notes cross-repo
  coordination, but this Atlas builder session is scoped to ATLAS only. The
  policy uses a portable contact shape so the companion repo can mirror it.
- This does not implement the M6 incident-response playbook or M9 CVE SLA.
  Those are adjacent policy slices with broader operational content.

## Deferred

- #1656 M6 incident response playbook, M9 CVE remediation SLA, M1 RLS, M2 JSONB
  encryption, M3 alert delivery, M5 scanner ratcheting, and H3 provider-side
  credential rotation remain separate slices.

Parked hardening: none.
