# PR-Gitignore-Secret-Env-Hardening
## Why this slice exists
Worktree cleanup found untracked env backup/export files that contain
secret-looking values and are not covered by the current ignore rules:
`.env.bak-preflag-20260523` and
`atlas-intel-ui/.atlas-intel-ui.production.env`. A blanket `git clean -fd`
would remove them, and a careless `git add` could stage similarly named files.
This workflow/process slice makes the durable repo rule explicit so future
cleanup and branch work treats those files as local secrets.
## Scope (this PR)
Ownership lane: workflow/process
Slice phase: Workflow/process
1. Ignore timestamped `.env.bak-preflag-*` local backup files.
2. Ignore `*.production.env` files generated by deployment tooling.
3. Do not move, copy, print, or delete any local secret-bearing files.
### Files touched
- `plans/PR-Gitignore-Secret-Env-Hardening.md`
- `.gitignore`
## Mechanism
The environment section of `.gitignore` already excludes `.env` and
`.env.local`. This PR adds two adjacent patterns:

```gitignore
.env.bak-preflag-*
*.production.env
```

The patterns are intentionally narrow. They cover the exact cleanup landmines
observed in this checkout without ignoring broad non-secret project artifacts.
## Intentional
- No `git clean` is run and no untracked env files are modified; the purpose is
  prevention, not local secret management.
- The pattern is not `.env.*`, because the repo may intentionally track or
  review non-secret env examples in other forms.
## Deferred
- Operator follow-up: rotate or delete any local secret values found during the
  cleanup investigation if they are no longer needed.
Parked hardening: none.
