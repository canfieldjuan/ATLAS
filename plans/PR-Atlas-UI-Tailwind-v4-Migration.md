# PR-Atlas-UI-Tailwind-v4-Migration

## Why this slice exists

Dependabot opened `#1787` to bump `atlas-ui` from Tailwind 3.4.19 to 4.3.1.
The manifest-only bump could not build because Tailwind v4 moved its PostCSS
plugin out of `tailwindcss` and removed the old `@tailwind base/components/
utilities` CSS entrypoint. The root cause was an incomplete major-version
migration: the package version changed, but the local PostCSS and CSS entrypoint
contracts still targeted Tailwind v3.

This exceeds the 400 LOC soft cap because the lockfile churn is generated by
the Tailwind 3 to 4 dependency graph and the required `@tailwindcss/postcss`
adapter. The hand-authored migration is limited to the PostCSS config, CSS
entrypoint, and this plan.

## Scope (this PR)

Ownership lane: dependency-maintenance
Slice phase: Production hardening

1. Keep Dependabot's `tailwindcss@4.3.1` upgrade for `atlas-ui`.
2. Add the Tailwind v4 PostCSS adapter package required by the existing Vite
   build path.
3. Migrate the `atlas-ui` CSS entrypoint from the removed Tailwind v3
   directives to the Tailwind v4 import.
4. Leave other UI packages and visual/source refactors out of this slice.

### Review Contract

Acceptance criteria:

- `atlas-ui/postcss.config.js` uses `@tailwindcss/postcss`, not `tailwindcss`
  directly, as the PostCSS plugin.
- `atlas-ui/src/index.css` no longer uses the removed Tailwind v3
  `@tailwind base`, `@tailwind components`, or `@tailwind utilities`
  directives.
- The local `atlas-ui` CI-equivalent path installs cleanly, reports no high
  severity npm audit findings, and builds.
- No package outside `atlas-ui` changes.

Affected surfaces:

- `atlas-ui` package dependency graph.
- `atlas-ui` PostCSS/Tailwind build path.
- `atlas-ui` global CSS entrypoint.

Risk areas:

- Tailwind v4 output differences could affect visual styling; this slice only
  proves the package build path, and preview/visual inspection can follow once
  CI is green.
- The shared Dependabot package plan no longer applies because this PR now
  includes source/config migration files.

## Mechanism

Tailwind v4 no longer allows PostCSS to load `tailwindcss` directly as the
plugin. The migration adds `@tailwindcss/postcss@4.3.1` and changes
`postcss.config.js` to load that adapter while leaving `autoprefixer` in place.

Tailwind v4 also replaces the old three-directive CSS entrypoint with a single
`@import "tailwindcss";`. `atlas-ui/src/index.css` keeps its existing body
styling after that import.

## Intentional

- The slice does not migrate other Tailwind packages; each Dependabot PR has its
  own package-specific blockers.
- No visual restyling is attempted here. The build path is the required fix for
  the security/dependency PR to become reviewable.
- `autoprefixer` remains in `postcss.config.js` because it was already part of
  the package pipeline and removing it is not required for the Tailwind v4
  repair.

## Deferred

- Visual preview smoke for `atlas-ui` after CI produces a clean preview signal.
- Separate Tailwind v4 migrations for `atlas-churn-ui` and `atlas-intel-ui` if
  their package-specific checks show equivalent blockers.
- `atlas-mobile` NativeWind/Tailwind migration remains separate because it is
  not safe as a dependency-only Tailwind v4 bump.

Parked hardening: none.
