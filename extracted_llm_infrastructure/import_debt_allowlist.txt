# Import-debt allowlist for the extracted_llm_infrastructure scaffold.

# The scaffold is a verbatim byte-snapshot of atlas_brain sources. The
# resolver in scripts/check_extracted_llm_infrastructure_imports.py
# (mirrored from the content-pipeline pattern) flags every relative
# import that does not literally resolve inside the scaffold's
# directory. Each entry below names a relative import target that lives
# in atlas_brain at the equivalent path, so this file documents which
# atlas_brain modules the scaffold currently leans on.
#
# Phase 2 (standalone substrate) landed five bridge stubs gated on
# EXTRACTED_LLM_INFRA_STANDALONE=1: config, services/{base,protocols,
# registry}, storage/database. The relative-import targets below still
# resolve through atlas_brain by default; with the toggle on they
# resolve to local _standalone/* copies instead.
#
# Phase 3 shrinks this list further by rewiring the scaffolded provider
# modules (services/llm/*, services/b2b/anthropic_batch.py,
# pipelines/llm.py, reasoning/semantic_cache.py, services/llm_router.py,
# services/tracing.py) to consume the standalone substrate via
# Protocol-based DI seams.
#
